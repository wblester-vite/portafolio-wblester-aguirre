Detección de Fraude en Transacciones Financieras

Tecnologías Utilizadas

🐍 Python | 📊 Pandas | 🔢 Scikit-learn | 📉 Matplotlib | 📌 Seaborn

🤖 Machine Learning | ⚙ Feature Engineering

- Descripción del Proyecto

Este proyecto tiene como objetivo construir un modelo de Machine Learning para detectar fraudes en transacciones financieras. Se realizó un análisis exploratorio de datos (EDA), selección de características relevantes y entrenamiento de un modelo de Regresión Logística. El modelo fue evaluado utilizando métricas clave como precisión, recall y curva ROC.

- Pasos del Proyecto

  • 📥 Carga y exploración de datos (EDA) para entender la distribución y patrones de las transacciones.
  • ⚙ Preprocesamiento y Feature Engineering: normalización de datos y selección de variables más influyentes.
  • 🤖 Entrenamiento y evaluación del modelo de Regresión Logística.
  • 📈 Análisis de resultados: evaluación con matriz de confusión y curva ROC.

- Resultados

El modelo de Regresión Logística logró una precisión del 99% en la detección de transacciones legítimas y un recall del 87% en transacciones fraudulentas. Se identificaron las características más influyentes en la detección de fraudes y se generó una matriz de confusión para evaluar su desempeño.

Detección de Fraude en Seguros con Machine Learning

Tecnologías Utilizadas

🐍 Python | 📊 Pandas | 🔢 Scikit-learn | 📉 Matplotlib | 📌 Seaborn

🤖 Machine Learning | ⚙ Feature Engineering | 📈 XGBoost | 🌲 Random Forest

- Descripción del Proyecto

Este proyecto tiene como objetivo construir un modelo de Machine Learning para detectar fraudes en reclamos de seguros. Se realizó un análisis exploratorio de datos (EDA), creación de variables derivadas y entrenamiento de dos modelos: Random Forest y XGBoost. Evaluamos la efectividad de los modelos con y sin la variable FRAUD_RATIO.

- Pasos del Proyecto

  • 📥 Carga y exploración de datos para entender patrones de fraude.
  • ⚙ Creación de variables derivadas como FRAUD_RATIO y segmentación de agentes sospechosos.
  • 🤖 Entrenamiento de dos modelos: Modelo 1 (con FRAUD_RATIO) y Modelo 2 (sin FRAUD_RATIO).
  • 📈 Evaluación y comparación de los modelos utilizando métricas clave como precisión, recall y matriz de confusión.

- Resultados

El modelo de Machine Learning basado en Random Forest y XGBoost logró detectar agentes con alta precisión. Se comparó el rendimiento con y sin la variable FRAUD_RATIO, concluyendo que aunque esta variable mejora la precisión, el modelo sigue siendo confiable incluso sin ella. Se identificaron factores clave que influyen en la detección de fraudes.
